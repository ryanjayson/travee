# Map & Sharing Feature Specification

## Overview
The application provides two complementary map experiences:

1. **MapViewer** — A full-screen interactive map modal (Mapbox GL JS in WebView) with activity pins, settings controls, and an overlay outline mode using a draggable/pinchable `CountryOutline` SVG.
2. **ShareTripModal** — A capturable share card (via `react-native-view-shot`) that renders either a static Mapbox map with pins or a `CountryOutline` SVG, composited with trip info text, activity emoji chips, and optional background image.

Both modals remain mounted; visibility is controlled via the `visible` prop.

## 1. MapViewer (`src/features/Travel/components/MapViewer/index.tsx`)

### Props
```typescript
interface MapViewerProps {
  visible: boolean;
  onClose: () => void;
  coordinates?: { latitude: number; longitude: number };
  title: string;
  zoom?: number;
  markers?: MapMarker[];
  destination?: string;
  countryName?: string;
}
```

### MapMarker Interface
```typescript
interface MapMarker {
  id?: string;
  latitude: number;
  longitude: number;
  title: string;
  type?: string | number;
  sortOrder?: string;
  images?: Array<{ url: string }>;
}
```

### Architecture: Dual Rendering

| Mode | `displayMode` | Renderer | Gestures |
|------|---------------|----------|----------|
| Map View | `"map"` | WebView with Mapbox GL JS (HTML written to `cacheDirectory/map.html`, loaded from `file://` URI) | Native Mapbox pan/zoom |
| Overlay Outline | `"overlay"` | `CountryOutline` SVG in `Animated.View` | Custom `PanResponder` + pinch-to-scale (0.3x–5x) |

### Map View (WebView)
- HTML generated by `generateMapHtml()` — static string embedded with marker data, pin sizes, map style, labels, etc.
- Written to disk via `expo-file-system` `File` API (`Paths.cache + "map.html"`)
- WebView keyed by `renderKey` (incremented on each rebuild) to force remount with fresh content
- Image pins: images are cached to `cacheDirectory/map-imgs/` as base64 files via `expo-file-system` `File.base64()` + `File.write(encoding: "base64")`
- Markers rendered as custom HTML elements (`.custom-marker` for type icons, `.image-marker` for image thumbnails)
- Labels rendered as `.marker-label` divs below the marker when `showLabels` is true
- Route line support removed (was implemented via GeoJSON `LineString` layer)

### Settings Panel (expandable floating bottom toolbar)

| Setting | Type | Values | Effect |
|---------|------|--------|--------|
| Filter Activities | Modal | Checkbox list per activity | Controls which markers appear (via `visibleIds: Set<string>`) |
| Pin Style | Segmented | `type` / `image` | Type icons vs image thumbnails on markers |
| Pin Size | Segmented | `small` / `medium` / `large` | 24/32/42px for type, 28/40/52px for image markers |
| Map Type | Segmented | `normal` / `satellite` / `hybrid` | Mapbox style URL (`streets-v12`, `satellite-v9`, `satellite-streets-v12`) |
| View | Segmented | `map` / `overlay` | Switches between WebView and CountryOutline |
| Show Labels | Switch | on/off | Toggles activity name badges below pins |

- Panel animates with `Animated.Value` — max height 420px.
- Collapsed pill shows current mode summary with active settings count badge.

### Build Effect (stabilized)
Two separate effects prevent infinite rebuild loops:

1. **VisibleIds Init Effect** — depends on `[visible, markersKey]`. Populates `visibleIds` Set from marker IDs on mount and when marker content changes.
2. **Build Effect** — depends on `[settingsKey, visibleIds, coordKey, markersKey, visible, displayMode]`. Uses content-based keys:
   - `settingsKey` = `JSON.stringify({ pinMode, pinSize, mapType, showLabels, displayMode, countryName, zoom })`
   - `coordKey` = stringified lat/lng
   - `markersKey` = joined string of `"id-lat-lng"` per marker
   
   Only runs when `displayMode === "map"`. Skips first mount until `visibleIds` is populated.

### Overlay Outline Mode
- Displays `CountryOutline` inside an `Animated.View` with `PanResponder` for drag and pinch-to-scale (two-finger)
- Layer selector at top-left: `[Outline]` / `[Image]` (only shown when image is attached)
- Active layer tracked via `activeLayerRef` to avoid stale closures in PanResponder
- Reset button resets position + scale for the currently active layer
- Image background: full-area `absolute inset-0`, `resizeMode="contain"`, no borders/styling
- Both layers (image + outline) have independent `translateX/Y` and `scale` animated values
- `outlineActivities` computed from `markers` filtered by `visibleIds`, mapped to `DoneActivity[]` type for CountryOutline
- Background: dark blue `#0C2A5A` with a bottom gradient overlay (`h-24 bg-black/30`)

### Activity Filter Modal
- Opened via "Filter Activities" button in settings
- Full list of activities with checkbox icons
- "Select All" and "Deselect All" buttons
- Counter badge: `{visibleIds.size}/{total}`
- Activities with `id` are toggleable; destination marker (no `id`) always visible.

## 2. ShareTripModal (`src/features/Travel/components/ShareOverlay/ShareTripModal.tsx`)

### Props
```typescript
interface ShareTripModalProps {
  visible: boolean;
  onClose: () => void;
  tripTitle: string;
  destination: string;
  countryName: string;
  dateRange?: string;
  activities?: ActivityPin[];
  doneActivities?: DoneActivity[];
}
```

### Modes
| Mode | `mapType` | Renderer | Details |
|------|-----------|----------|---------|
| Map View | `"map"` | `Image` with Mapbox Static API URL | Shows pins + straight-line path between sorted activities |
| Outline | `"outline"` | `CountryOutline` SVG (inside `Animated.View`) | Country shape with activity dots |

### Map View Implementation
- Static API URL: `https://api.mapbox.com/styles/v1/mapbox/streets-v12/static/{pins}{path}/auto/600x400@2x`
- Pins: `pin-{s|m|l}+{hexColor}(lng,lat)` for each activity
- Path: `path-3+{hexColor}-0.5(lng1,lat1;lng2,lat2;...)` connecting activities sorted by `sortOrder`
- Auto-fit ensures all pins are visible

### Share Card Layout (capturable via ViewShot)
1. Background: dark blue `#0C2A5A` with gradient overlay
2. Optional user-picked background image (via `expo-image-picker`)
3. Draggable/scalable content (CountryOutline or static map Image) via `Animated.View` with `PanResponder` + pinch
4. Draggable trip info block (TRAVIE branding, title, destination, emoji chips, date range)
5. Share + Save buttons at bottom

### Pin Settings
- **Color**: 8 preset colors (red, blue, green, orange, purple, teal, crimson, dark grey)
- **Size**: S / M / L (maps to `pin-s`, `pin-m`, `pin-l` in Static API)
- Settings panel toggled via palette icon button

## 3. CountryOutline (`src/features/Travel/components/ShareOverlay/CountryOutline.tsx`)

### Props
```typescript
interface CountryOutlineProps {
  countryName: string;
  width: number;
  height: number;
  strokeColor?: string;      // default '#ffffff'
  strokeWidth?: number;      // default 1.5
  doneActivities?: DoneActivity[];
  pinSize?: 'small' | 'medium' | 'large';   // default 'medium'
  showLabels?: boolean;                       // default false
}

interface DoneActivity {
  lat: number;
  lng: number;
  type?: number;
  title?: string;
}
```

### Implementation
- Loads `assets/geo/countries.json` (bundled GeoJSON) via `require`
- Uses `d3-geo` `geoMercator().fitExtent()` to project the country outline onto canvas dimensions
- `geoPath().projection(projection)` generates the SVG `d` path data
- Activity pins mapped from lat/lng to pixel coordinates via the same projection
- Pin dot radius: small=2, medium=3, large=5 (stored in `PIN_SIZE_R` lookup)
- Optional SVG text labels below pins (white text with subtle dark stroke for readability)
- Legend chips (top activity types) currently commented out

## 5. Data Flow & Efficiency

### WebView Image Loading
1. `expo-image-picker` returns `file://` URIs
2. `expo-file-system` `File.base64()` reads the file
3. Written to `cacheDirectory/map-imgs/` with `File.write(encoding: "base64")`
4. WebView loaded from `cacheDirectory/map.html` (`file://` URI) with `allowFileAccess` enabled
5. Images referenced as `file://` paths to cache directory

### Stable Effect Keys (prevents infinite reload loops)
- `settingsKey`: JSON-stringified settings object
- `coordKey`: stringified coordinates
- `markersKey`: joined content-based marker identifiers
- These ensure the build effect only runs when meaningful data changes, not on every parent re-render.

### Memory
- Only ONE country's GeoJSON is embedded (via `countriesGeoJSON.features.find()`)
- Images are cached once (`destFile.exists` check), reused across rebuilds
- Counter-based naming (`imgCounter++`) prevents collisions
